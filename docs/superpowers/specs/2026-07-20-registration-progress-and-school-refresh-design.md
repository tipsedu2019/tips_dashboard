# 등록 진행 스텝퍼·학교 갱신·레벨테스트 장소 설계

**작성일:** 2026-07-20  
**상태:** 승인  
**관련 설계:** `2026-07-20-registration-application-layout-and-subject-tabs-design.md`

## 1. 변경 목적

등록 신청서는 한 장의 공통 문서이고 영어·수학 트랙은 독립적으로 진행한다. 이번 변경은 다음 네 가지 운영 문제를 해결한다.

1. 환경설정에서 학교를 추가한 직후 등록 신청서가 오래된 학교 목록을 보여 주는 문제를 없앤다.
2. 현재 과목의 전체 등록 흐름을 신청서 상단에서 한 번에 파악하게 한다.
3. 입학 처리 다섯 단계를 실제 진행 순서대로 보여 주고 완료 여부를 체크 상태로 구분한다.
4. 레벨테스트 장소의 신규 입력을 `본관` 또는 `별관` 중 하나로 제한한다.

## 2. 학교 목록 최신화

`academic_schools`에는 `기타`가 `elementary` 분류로 정상 저장되어 있다. 누락 원인은 분류 필터가 아니라 사용자별 등록 옵션 캐시에 만료·무효화 경계가 없는 데 있다.

- 등록 신규·상세·과목 작업·예약 작업을 열 때 등록 옵션을 강제로 다시 조회한다.
- `기타`는 학교명으로 취급한다. 학교급 별칭으로 추가하지 않는다.
- 학년별 학교급 필터, 정렬, 기존 저장값 보존 규칙은 유지한다.
- 재조회 실패 시 기존의 오류·재시도 계약을 유지한다.

## 3. 상단 과목별 진행 스텝퍼

저장된 신청서는 기존 과목 탭의 선택 트랙을 기준으로 다음 다섯 단계를 상단에 표시한다.

1. 문의
2. 레벨테스트
3. 상담
4. 등록·대기
5. 입학 처리

- 과목 탭과 스텝퍼를 헤더 아래, 문의 정보 위에 둔다.
- 과목 탭을 바꾸면 같은 `activeTrackId`에서 파생한 진행상황이 함께 바뀐다.
- 신규 신청서는 아직 트랙이 없으므로 `문의`를 현재 단계로 표시한다.
- 단계 상태는 `지남`, `현재`, `예정`, `완료`, `종료`를 색 외에 아이콘·텍스트·`aria-current="step"`로 구분한다.
- 레벨테스트나 상담을 건너뛸 수 있으므로 앞선 모든 단계를 임의로 `완료`라고 표시하지 않는다.
- `registered`만 전체 완료로 표시한다. `not_registered`, `inquiry_closed`는 해당 위치의 종료로 표시한다.
- 본문의 일반적인 `진행상태`, `첫 저장 전`, 반복되는 `진행 중` 표시는 제거한다.
- 시험 시작·완료 상태, 등록 단계처럼 실제 업무 사실을 나타내는 필드는 유지한다.

## 4. 입학 처리 스텝퍼

입학 처리는 신청 건 단위로 한 번만 렌더링하고 다음 순서를 하나의 세로 목록으로 표시한다.

1. 입학신청서 발송
2. 메이크에듀 등록(수업, 교재)
3. 청구서 발송
4. 수납 완료 확인
5. 등록 완료

- `<ol aria-label="입학 처리 진행">` 안에 다섯 단계만 둔다.
- 각 단계는 체크 표시, 순번, 이름, 현재 상태, 기존 실행 컨트롤을 함께 가진다.
- 첫 미완료 단계를 현재 단계로 표시한다.
- 체크는 서버의 권위 상태를 반영한다. 외부 메시지를 보내는 1단계를 단순 체크박스 토글로 바꾸지 않는다.
- 2단계 안에 입학 처리 대상 선택, 처리 시작, 과목·수업별 메이크에듀 등록 컨트롤을 유지한다.
- 3~5단계의 기존 RPC와 선행 조건을 그대로 사용한다.
- 신규 신청서에는 같은 다섯 단계의 잠긴 미완료 상태를 표시한다.

## 5. 레벨테스트 장소

- 신규 레벨테스트 장소는 기본값 없는 단일 선택으로 표시한다.
- 선택 가능한 값은 정확히 `본관`, `별관`이다.
- 저장된 레벨테스트 예약 편집기도 동일하게 제한한다.
- 방문상담 장소는 현재 자유 입력을 유지한다.
- 조회 모델과 이력의 `place: string` 타입은 유지하여 `본관 201호` 같은 기존 기록을 손실하지 않는다.
- 기존 비표준 레벨테스트 장소는 읽기 전용 화면에서 원문을 보여 준다. 수정 화면에서는 기존 값을 안내하고 `본관` 또는 `별관`을 다시 선택해야 저장할 수 있다.
- 신규 작성과 정상 예약 저장 경로에서 비표준 값을 차단한다. 마이그레이션·이력 조회 값은 바꾸지 않는다.

## 6. 접근성과 반응형

- 전체 진행은 의미 있는 ordered list와 `aria-current="step"`를 사용한다.
- 과목 탭은 기존 키보드 화살표 이동을 유지한다.
- 데스크톱에서는 전체 진행 스텝퍼를 가로로, 작은 화면에서는 세로 또는 가로 스크롤 없이 읽을 수 있는 압축 목록으로 표시한다.
- 입학 처리 스텝퍼는 화면 크기와 무관하게 세로 순서를 유지한다.
- 완료·현재·예정은 색만으로 구분하지 않는다.

## 7. 완료 조건

- 학교 설정에서 저장한 `초등 > 기타`가 페이지 새로고침 없이 다음 등록 진입에서 보인다.
- 영어·수학 탭 전환 시 상단 진행상황도 해당 과목으로 바뀐다.
- 일반적인 진행상태 문구가 본문에 중복 노출되지 않는다.
- 입학 처리 다섯 단계가 한 번만, 순서대로, 체크 상태와 기존 실행 행동을 포함해 표시된다.
- 레벨테스트 신규·수정 입력은 `본관`, `별관`만 허용하고 방문상담 및 기존 이력은 손상되지 않는다.
- 실제 공급자 알림을 보내지 않은 상태에서 단위·계약·브라우저 검증이 통과한다.
