# 과학 과목 기반 및 운영 확장 설계

**작성일:** 2026-07-22
**상태:** 사용자 권장안 승인 완료
**대상 저장소:** TIPS Dashboard 관리자 앱

## 1. 목표

영어와 수학에 이어 과학을 독립적인 정규 과목으로 추가한다. 과학은 별도 과학 원장과 과학팀이 운영하며, 등록은 영어·수학과 같은 subject-track 프로세스를 사용한다. 레벨테스트는 건별로 실시하거나 생략할 수 있다.

이번 변경은 과학 기능을 화면마다 직접 덧붙이는 패치가 아니다. 과목의 안정적인 정체성은 타입과 코드 정책으로 고정하고, 운영 중 바뀔 수 있는 활성 상태·학년·담당 원장·강의실·세부과목·알림 연결은 DB 설정으로 관리하는 하이브리드 기반을 만든다.

완료 상태는 다음을 모두 만족해야 한다.

- 과학은 영어·수학과 구분되는 독립 과목이다.
- 과학 신규 운영 대상은 고1·고2·고3뿐이다.
- 과학 세부과목은 통합과학, 물리학, 화학, 생명과학, 지구과학이다.
- 과학 등록은 문의, 선택적 레벨테스트, 상담, 대기, 등록, 완료 흐름을 사용한다.
- 과학 원장과 과학팀을 영어·수학 조직과 별도로 선택할 수 있다.
- 강의실은 중복 생성하지 않고 `subjects[]` 다중 소속을 사용하며, 초기 운영에서는 기존 `별관 4강`에 과학을 추가한다.
- 학사일정과 연간보드에서 과학 시험일과 시험범위를 관리한다.
- 교재에서 과학과 5개 세부과목을 관리한다.
- 과학 전용 Google Chat 연결 항목을 만들되 외부 실발송은 계속 꺼 둔다.
- 과학에는 영어 단어 재시험, 과학 전용 월간보고·전자결재 템플릿을 추가하지 않는다.
- 공개 홈페이지와 홍보 화면은 다른 저장소의 범위이므로 수정하지 않는다.

## 2. 비목표

- 관리자가 임의의 네 번째 과목을 한 번의 DB 행 추가만으로 생성하는 범용 과목 플랫폼
- 기존 영어·수학 등록 상태머신의 재설계
- 영어 단어 재시험을 과학에 일반화하는 작업
- 과학 전용 월간보고 또는 전자결재 양식
- 공개 홈페이지의 문구, 수업 목록, SEO 변경
- 알림 provider 활성화, 실제 Google Chat·Web Push·SOLAPI 발송
- 측정 근거 없는 전역 성능 리팩터링
- 기존 `기타` 교재를 과학으로 일괄 변환하는 작업

## 3. 핵심 설계 원칙

### 3.1 고정 과목 코어

앱은 다음 3개 과목만 정식 과목으로 인식한다.

| 키 | 표시값 | 팀 | 정렬 |
| --- | --- | --- | ---: |
| `english` | 영어 | 영어팀 | 10 |
| `math` | 수학 | 수학팀 | 20 |
| `science` | 과학 | 과학팀 | 30 |

공통 과목 모듈은 다음 책임만 가진다.

- 키와 표시값의 양방향 변환
- 별칭 정규화
- 안정적인 정렬
- 지원 과목 검증
- 과목별 명시적 기능 정책

알 수 없는 값은 영어로 바꾸지 않는다. 읽기 경계에서는 `null` 또는 명시적 오류로 닫고, 쓰기 경계에서는 저장을 거부한다. 기존 데이터에 저장된 정식 과목은 신규 생성 활성 여부와 무관하게 계속 읽을 수 있어야 한다.

### 3.2 과목별 기능 정책

공통 모듈에는 다음 capability를 명시한다.

| capability | 영어 | 수학 | 과학 |
| --- | --- | --- | --- |
| 등록 subject track | 지원 | 지원 | 지원 |
| 선택적 레벨테스트 | 지원 | 지원 | 지원 |
| 방문·전화상담 | 지원 | 지원 | 지원 |
| 수업·시간표 | 지원 | 지원 | 지원 |
| 교재·세부과목 | 지원 | 지원 | 지원 |
| 학교 시험일·시험범위 | 지원 | 지원 | 지원 |
| 과목팀 Google Chat 연결 | 지원 | 지원 | 지원 |
| 영어 단어 재시험 | 지원 | 미지원 | 미지원 |
| 과목 전용 월간보고 | 기존 영어 정책 | 기존 수학 정책 | 미지원 |

기능이 없는 과목을 조건문의 `else`로 다른 과목에 흡수하지 않는다.

### 3.3 DB 운영 설정

`academic_subject_settings`는 임의 과목 생성 테이블이 아니라 고정 3과목의 운영 설정이다.

- `subject`: 영어·수학·과학 중 하나인 PK
- `is_active`: 전체 운영 활성 상태
- `registration_create_enabled`: 신규 등록 선택 허용 여부
- `grade_levels`: 허용 학년 배열
- `default_director_profile_id`: 기본 과목 원장
- `sort_order`
- 감사용 `created_at`, `updated_at`

초기값은 영어·수학의 기존 운영을 보존하고, 과학은 활성·등록 생성 가능·고1~3으로 설정한다. 과학 기본 원장이 아직 선택되지 않아도 문의 또는 레벨테스트 경로는 저장할 수 있다. 전화상담이나 방문상담처럼 원장이 필요한 경로는 과학 원장 선택 전까지 fail-closed로 막는다.

인증 사용자는 안전한 읽기 RPC로 설정을 읽는다. 직접 쓰기는 운영자만 가능하다. RPC가 아직 없는 구버전 DB에서는 영어·수학만 노출하고 과학 신규 쓰기는 숨긴다.

과학 세부과목은 표시 문자열만 저장하지 않고 `academic_subject_areas`의 안정 키를 사용한다.

| 안정 키 | 표시명 |
| --- | --- |
| `integrated_science` | 통합과학 |
| `physics` | 물리학 |
| `chemistry` | 화학 |
| `life_science` | 생명과학 |
| `earth_science` | 지구과학 |

안정 키와 subject 연결은 변경할 수 없다. 표시명, 정렬, 활성 상태는 운영 설정으로 바꿀 수 있지만 활성 수업·교재·시험 이력이 참조하는 항목은 삭제하지 않는다.

## 4. 조직과 권한

선생님 설정의 팀 목록에 `과학팀`을 추가하고 가입 metadata 정규화에서도 과학팀을 영어팀으로 잘못 바꾸지 않는다. 시간표의 학술팀 필터에도 과학팀을 추가한다.

기본 과학 원장은 `academic_subject_settings.default_director_profile_id`로 지정한다. 현재 `profiles`에는 별도 활성 컬럼이 없으므로 지정 대상은 존재하는 profile, 삭제·정지되지 않은 auth 계정, `teacher_catalogs.profile_id`로 연결된 보이는 교사 카탈로그, `과학팀` 소속을 모두 만족해야 한다. 과학 원장은 전역 `admin` 역할을 강제로 부여받지 않고 기존 `teacher` 역할을 유지할 수 있다. `is_active_subject_director(profile_id, subject)`가 과학 설정과 계정·팀 소속을 확인하여 과학 트랙에만 subject-scoped 원장 권한을 부여한다. 등록 트랙에는 실제 배정된 원장 profile ID를 역사 스냅샷으로 저장한다.

현재 영어·수학 원장의 기존 자동 배정 규칙은 유지한다. 과학은 고정된 사람 이름을 코드나 migration에 넣지 않고 DB 설정으로 결정한다. 기본 원장이 없거나 계정 연결이 유효하지 않으면 자동 배정하지 않는다.

이번 변경은 전역 역할 체계를 다시 만들지 않는다. 등록 상담 완료 권한은 기존처럼 배정된 track director 본인과 운영자에게만 허용하며, 과학 원장이라는 이유만으로 다른 과목 트랙이나 전역 알림 설정을 변경할 수 있게 하지 않는다.

## 5. 등록

### 5.1 생성과 편집

- 신규 등록 과목 선택기는 DB capability와 학년을 함께 확인한다.
- 과학은 `고1`, `고2`, `고3`에서만 선택할 수 있다.
- 과학을 선택한 뒤 학년을 중등·초등으로 바꾸면 과학을 자동으로 조용히 유지하지 않고 선택에서 제거하며 화면에 이유를 표시한다.
- 기존 과학 트랙은 나중에 capability를 끄더라도 읽기·이력 화면에서 사라지지 않는다.
- 한 학생의 영어·수학·과학은 하나의 parent case 아래 독립 track으로 저장한다.
- 최대 track 수는 3개다.

### 5.2 초기 경로

각 과목은 다음 중 하나를 독립적으로 선택한다.

- 문의 유지
- 레벨테스트
- 바로 전화상담
- 방문상담

과학도 같은 네 경로를 지원한다. 레벨테스트와 방문상담의 공유 예약에는 최대 3개 과목이 참여할 수 있다. 시험 결과 URL, 상담 결과, 등록 수업과 교재는 track별로 저장한다.

### 5.3 호환성과 capability gate

기존 등록 runtime marker의 반환값은 바꾸지 않는다. 별도의 `list_registration_subject_capabilities_v1()` 읽기 RPC가 신규 과목 생성 가능 여부를 제공한다.

배포 호환 순서는 다음과 같다.

1. 과학을 안전하게 읽되 capability RPC가 없으면 신규 선택을 숨기는 앱
2. 고정 3과목 설정·constraint·RPC를 추가하는 additive migration
3. 과학 운영 설정과 담당자 확인
4. 과학 신규 등록 UI 활성 확인

DB migration은 기존 공개 RPC signature를 깨지 않는다. 과목 검증, 최대 개수, 정렬, 공유 예약 cardinality, parent projection을 최신 함수 정의 기준으로 확장한다.

## 6. 수업·강의실·시간표

### 6.1 수업

수업의 정식 과목 목록에 과학을 추가한다. 과학 수업은 고1~3만 허용하고 다음 세부과목 중 하나를 필수로 가진다.

- 통합과학
- 물리학
- 화학
- 생명과학
- 지구과학

`classes.subject_area_key`는 다른 과목에서는 비워 두고 과학에서는 필수다. 값은 활성 과학 세부과목 안정 키 중 하나여야 한다. DB constraint와 저장 서비스가 동일한 규칙을 검증한다.

### 6.2 강의실

`classroom_catalogs.subjects[]`가 authoritative하다. 설정 화면은 단일 select를 다중 선택으로 바꾸고 기존 배열의 다른 과목을 보존한다.

초기 데이터 migration은 이름이 정확히 `별관 4강`인 기존 행에 과학을 추가한다. 새 강의실 행을 만들지 않는다. 해당 행이 없는 환경에서는 migration을 실패시키지 않고 설정 화면에서 운영자가 지정할 수 있게 한다.

### 6.3 시간표

시간표 과목 필터와 교사·강의실 후보 필터에 과학을 추가한다. 과학 수업에는 과학팀 선생님과 과학을 허용한 강의실만 후보로 표시한다. 영어+과학 공유 강의실은 두 과목 모두에서 보인다.

## 7. 교재와 세부과목

교재 subject enum에 `science`를 추가하고 `과학` 별칭을 같은 값으로 정규화한다. 기존 unknown 값의 `other` fallback은 읽기 호환에만 사용하며, 정식 저장 입력의 unknown 값은 거부한다. 과학 교재는 `subject_area_key`를 필수로 저장하고 기존 `sub_subject`에는 해당 안정 키의 현재 표시명을 호환 projection으로 유지한다.

과학 기본 세부과목 설정은 다음 순서로 seed한다.

1. 통합과학
2. 물리학
3. 화학
4. 생명과학
5. 지구과학

과학 교재는 `high`와 `h1`·`h2`·`h3`만 허용한다. 과학을 선택하면 입력 UI가 학교급과 학년을 이 범위로 정규화한다. 기존 `기타` 교재는 자동 변환하지 않는다.

교재 재고·구매·판매·마감은 기존 공통 ledger를 사용한다. 과학팀 마진 집계가 필요한 공통 집계에서는 `science` 버킷을 독립적으로 유지하고 `other`로 합치지 않는다.

## 8. 학사일정과 대시보드

연간보드 type에 `과학시험일`을 추가하고 고등학교 행에서만 생성·편집한다. 과학 시험 셀은 기존 영어·수학과 같은 방식으로 다음을 표시한다.

- 시험일
- 시험범위
- 연결 교재·부교재 범위
- 미입력 경고

월간 학사 캘린더와 대시보드 시험 충돌 계산도 `과학시험일`을 정식 시험 이벤트로 인식한다. 중등·초등 학교 행에는 과학 시험 생성 action을 표시하지 않는다.

## 9. 알림

알림 connection key에 `google_chat.science`를 추가하고 설정 화면에는 `과학팀 Google Chat`으로 표시한다. DB connection channel enum과 안전 DTO, worker target validation, subject-team connection resolver도 과학을 명시적으로 지원한다.

신규 연결은 다음 안전 상태로 생성한다.

- webhook secret 없음
- disconnected
- runtime enabled 아님
- 실제 규칙 활성 상태 변경 없음
- provider 호출 없음

등록 알림 adapter는 과학과 3과목 payload를 읽고 정렬할 수 있어야 한다. 알 수 없는 과목은 계속 거부한다. 기존 `[팁스영어수학학원]` 고객 메시지 브랜드 문구는 이 저장소에서 변경하지 않는다.

## 10. 명시적 제외 경계

다음 화면과 모델은 과학을 추가하지 않는 것이 정상이다.

- 영어 단어 재시험 생성·필터·자동 미응시·재재시험
- 영어 전용 주간시험 및 오답 흐름
- 과학 전용 월간보고 템플릿
- 과학 전용 전자결재 템플릿
- 공개 홈페이지와 landing 문구

일반 전자결재 기능은 과목과 무관하게 계속 사용할 수 있다.

## 11. 오류 처리와 롤백

- 앱 parser가 unknown subject를 만나면 영어로 보정하지 않고 명시적 준비 오류를 표시한다.
- capability RPC 오류는 과학 신규 선택만 닫고 기존 영어·수학 등록을 막지 않는다.
- science track read는 활성화 여부와 무관하게 허용한다.
- 잘못된 학년, 세부과목, 담당자, 공유 예약 membership은 DB RPC에서도 재검증한다.
- 과학 운영을 긴급 중단할 때는 `registration_create_enabled=false`로 신규 쓰기만 막는다.
- 이미 저장된 과학 데이터, 확장 constraint, 이력은 롤백 과정에서 삭제하지 않는다.
- 알림은 connection과 provider runtime flag를 별개로 유지한다. 과학 connection을 추가해도 발송은 자동 활성화되지 않는다.

## 12. 저장소 위생과 성능

Codex가 표시한 수백만 줄은 앱 소스가 아니라 저장소 안의 `.pnpm-store` 캐시다. `.gitignore`에 `/.pnpm-store/`를 추가하되 캐시나 기존 worktree를 삭제하지 않는다.

성능 작업은 다음으로 제한한다.

- 과목 목록과 capability를 중앙 모듈에서 한 번 계산한다.
- 화면별로 과목 설정을 추가 조회하는 N+1을 만들지 않는다.
- 기존 데이터 fetch에 새로운 `select("*")`를 추가하지 않는다.
- 직접 수정하는 대형 workspace에는 과목 규칙을 중복 작성하지 않고 작은 helper를 import한다.
- 변경 전후 전체 테스트, lint, production build 시간을 기록한다.

대형 workspace 전체 분해와 광범위한 렌더링·쿼리 최적화는 과학 기능과 별도의 측정 기반 작업으로 남긴다.

## 14. 완료 기준

- 코드, DB migration, 자동화 테스트, 로컬 build가 모두 통과한다.
- 과학이 고1~3에서만 신규 선택된다.
- 과학 등록의 레벨테스트 실시·생략 경로가 모두 보존된다.
- 별관 4강을 영어와 과학이 함께 선택할 수 있다.
- 과학 5개 세부과목이 수업과 교재 입력에서 사용된다.
- 과학 시험일·범위가 연간보드와 캘린더에 표시된다.
- 과학 Google Chat connection은 설정 가능하지만 실발송은 꺼져 있다.
- 단어 재시험과 과학 전용 월간보고에는 과학이 추가되지 않는다.
- 기존 dirty word-retest 변경을 덮어쓰지 않는다.
- 공개 홈페이지 파일을 수정하지 않는다.
