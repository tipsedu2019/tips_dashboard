# 등록 과학 선택 순서 독립화 설계

## 목표

등록 화면에서 학년을 고르기 전에도 과학을 영어·수학과 동일하게 선택할 수 있게 한다.

## 동작

- 등록 과목을 고르는 순서는 학년 선택 순서와 독립적이다.
- 학년이 비어 있으면 활성화된 모든 등록 과목을 선택할 수 있다.
- 학년을 선택하거나 바꾸면 서버에서 조회한 과목별 `gradeLevels`를 기준으로 지원하지 않는 과목을 제거한다.
- 학년 제한 문구에 `과학은 고등부만` 같은 현재 운영 범위를 하드코딩하지 않는다.
- 현재 운영 가능 학년은 과목 설정 데이터가 결정한다. 이번 변경은 선택 순서만 분리하며, 실제 초중등 과학 개방은 과목 설정 파서·수업 분류·DB 검증을 함께 변경하는 별도 작업이다.

## 변경 범위

`registrationSubjectDisabledReason`은 과목 활성화 여부를 먼저 검사한다. 학년이 비어 있으면 빈 사유를 반환하고, 학년이 채워진 경우에만 `capability.gradeLevels`를 검사한다. 과학만을 위한 선택 전 차단 로직을 제거한다.

통계, 전반, 퇴원, 휴보강의 과목 목록은 이번 변경에서 제외한다. DB 마이그레이션, 과목 운영 설정, 알림 발송 설정도 변경하지 않는다.

## 안전장치

- 학년은 등록 저장의 공통 필수값이므로 학년 없이 저장되지는 않는다.
- 학년 변경 시 지원하지 않는 기존 선택 과목과 해당 초기 상담 계획을 정리하는 현재 동작을 유지한다.
- 초중등 과학의 실제 허용 시점은 `RegistrationSubjectCapability.gradeLevels`와 DB 검증이 결정한다.
- 현재 과학 capability 파서와 DB의 고1~고3 fail-closed 제한은 유지한다.
- Google Chat, Web Push, SOLAPI 발송 설정은 그대로 둔다.

## 검증

- 학년 미선택 상태에서 과학이 활성화되는 단위 테스트.
- 빈 학년 상태의 정합성 처리에서 과학 계획과 원장 지정이 보존되는 단위 테스트.
- 지원하지 않는 학년을 선택하면 과학과 해당 계획이 제거되는 기존 회귀 테스트.
- 등록 브라우저 검증 스크립트에서 학년 선택 전에 과학을 눌러 선택 상태를 확인.
- 관련 테스트, TypeScript 검사, ESLint, `next build --webpack`, 등록 화면 브라우저 확인.
