# 등록 메뉴 브라우저 QA·UI 폴리싱 기록

- 작업일: 2026-07-19
- 대상: `/admin/registration`
- 작업 시작 기준: `main`의 `fbcf3006657ed00f22ad7b6e74779ed4370fa9aa`
- 상태: 실제 브라우저 QA, 수정, 재검증 완료

## 안전 조건

- 운영 배포 화면은 조회와 화면 전환만 확인했다. 운영 데이터의 저장·수정·상담 완료·입학 처리·고객 메시지 발송은 실행하지 않았다.
- 실제 저장 흐름은 개발 환경에서만 활성화되는 `fixture=registration-subject-tracks` 메모리 fixture로 검증했다.
- 로컬 3001 서버는 Google Chat, Web Push, SOLAPI, 알림 worker 관련 환경값을 빈 값으로 덮어쓴 프로세스로 실행했다. `.env.local`은 변경하지 않았다.
- 서버 접근 로그에서 등록 QA 동안 공급자 발송·알림 dispatch POST 요청은 0건이었다. 전역 셸의 Push 준비 상태 조회 GET만 발생했으며 권한 요청이나 발송은 없었다.
- fixture 요청 시 adapter가 아직 준비되지 않았거나 URL 전환으로 해제된 순간에는 실제 등록 서비스로 대체 실행하지 않고 닫힌 오류로 끝난다. 이 구간에는 공급자 토큰도 전달하지 않는다.
- 브라우저 콘솔의 오류·경고는 최종 확인 시 0건이었다.

## 실제 브라우저에서 확인한 운영 흐름

다음 화면을 데스크톱 1440px·1024px·820px, 모바일 390px·320px에서 확인했다.

1. 신규 문의 등록
   - 학생·보호자·학교·학년·캠퍼스·문의 시각·요청 사항의 필드 순서와 필수값 안내
   - 영어 또는 수학 단일 과목 등록
   - 영어·수학 복수 과목의 독립된 단계·담당자·예약 정보 저장
2. 레벨테스트
   - 예약, 응시 시작, 시험지·결과 URL 입력, 완료, 미응시, 재시험 예약 입력
   - 레벨테스트 완료 뒤 전화상담으로 이어지는 단계 이동
3. 전화·방문상담
   - 전화상담 예약·완료
   - 방문상담 일시·장소 예약·완료
   - 역할별 액션 노출과 상담 결과 저장 중 중복 클릭 잠금
4. 등록 결정과 대기
   - 등록 결정, 미등록 사유, 등록 취소 후 이동 단계
   - 현재 반 대기와 수업 선택값 저장·재열기
   - 다음 개강 알림 대기 저장·재열기
   - 종료된 상담 흐름을 이전 운영 단계로 다시 열기
5. 후속 입학 처리
   - 공통 입학 정보 수정
   - 입학원서 전달 상태, 과목별 등록 선택, 입학 처리 시작
   - 영어·수학 MakeEdu 등록, 청구서, 결제 확인, 등록 완료
   - 진행 중 등록 취소와 다음 개강 알림 전환
6. 목록·달력·상세
   - 검색, 상태 탭, 검색 결과 없음, 업무 없음, 로딩 상태
   - 월간·주간 달력, 예약 상태 필터, 달력 일정에서 상세·예약 편집기 연결
   - 목록과 달력 동시 새로고침, 선택 탭 유지
   - 자동 이력의 담당자·시간 자동 기록과 읽기 전용 표시
7. 실패와 복구
   - 강제 저장 실패 뒤 같은 팝업에서 재시도
   - 3초 지연 저장 중 중복 클릭
   - 상세·예약 로딩 실패와 재시도
   - 저장 뒤 재열기·새로고침 시 상태 일치

레벨테스트 재시험 예약은 네이티브 확인창 노출과 모든 입력까지 실제 브라우저로 확인했다. Chrome 제어 계층에서 네이티브 확인창 자동 승인이 멈추는 제약이 있어 최종 mutation은 동일 fixture 저장 계약과 등록 회귀 테스트로 추가 검증했다.

## 발견하고 수정한 문제

### 목록과 반응형

- 820px 표 압축과 내용 겹침을 없애기 위해 `lg` 미만은 모바일 카드, 충분한 폭에서만 표를 사용한다.
- 모바일 상태 탭을 한 줄 가로 스크롤로 바꿔 문의 → 레벨테스트 → 상담 → 대기 → 등록 → 완료 순서를 유지한다.
- 학생명, 과목, 상담 책임자, 학교·학년, 전화번호, 요청 사항을 함께 찾는 검색을 추가했다. 공백과 전화번호 하이픈 차이를 무시한다.
- 달력에서는 동작하지 않는 검색을 숨기고 목록·달력을 함께 갱신하는 새로고침을 추가했다.
- 관리자 행의 `상세`와 단계별 `관리` 중복을 제거하고 현재 업무 액션 하나만 남겼다.
- 검색 결과 없음과 현재 단계 업무 없음 문구를 분리하고 등록 전용 로딩 상태를 추가했다.
- 긴 학생명·담당자·장소·수업명을 말줄임으로 숨기지 않고 줄바꿈한다.

### 추가·수정·상세·팝업

- 불필요한 수동 설명 팝업과 중복 단계 안내를 제거했다.
- 320px에서 이전·다음·저장 버튼이 안전하게 줄바꿈하고, 하단 저장 영역이 입력 필드를 덮지 않도록 정상 문서 흐름으로 바꿨다.
- 닫기·취소 뒤 검증 오류와 완료 차단 문구가 목록에 남지 않도록 초기화했다.
- 상담 결과 팝업을 모바일 2열·데스크톱 3열로 조정하고, 긴 화면은 팝업 내부에서 스크롤한다.
- 수업, 대기 종류, 취소 후 단계 선택 상자에 과목별 접근성 이름과 안전한 입력 폭을 추가했다.
- 예약 편집기를 자동 이력보다 먼저 배치하고, 예약 관리 진입 시 편집 위치로 자동 스크롤한다.

### 상태 일치·재시도·fixture 정확성

- 목록의 방문상담 일시·장소가 `미정`으로 보이던 fixture 투영 누락을 수정했다. 수정·취소·완료 시 오래된 값도 제거된다.
- 현재 반 대기 수업이 재열기 뒤 비어 보이던 문제를 수정했다. 실제 등록 claim과 fixture가 같은 규칙으로 수업을 복원한다.
- 등록 결정 변경, 개별 수강 취소, 입학 처리 묶음 취소, 마이그레이션 보정에서도 현재 반 대기 claim을 운영 규칙과 똑같이 만들도록 fixture를 맞췄다. 기존 계획 수업은 취소 이력으로 남기고 활성 대기 수업과 중복되지 않는다.
- 강제 fixture 오류의 내부 코드가 그대로 보이지 않도록 `등록을 저장하지 못했습니다. 잠시 후 다시 시도해 주세요.`로 변환했다.
- 상세와 예약의 실패 상태를 분리하고 각각 올바른 재시도 동작을 연결했다.
- 저장 중 상담 결과·선택·닫기·ESC를 잠가 중복 mutation을 막았다.
- fixture용 재열기와 지연·1회 실패 제어는 개발·테스트의 정확한 fixture 주소와 허용된 액션에서만 동작하도록 제한했다.
- fixture 준비 전 화면은 로딩 상태로 잠그고, fixture에서 실패한 방문상담 알림 재시도 대상은 fixture 종료 시 제거해 운영 공급자 재시도와 섞이지 않게 했다.
