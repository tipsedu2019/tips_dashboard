# 알림 업무 전환·롤백 운영 절차

이 문서는 공통 알림 코드와 미리보기 검증이 끝난 뒤 사용할 운영 인계서다. 검증 스크립트는 읽기 전용이며 운영 플래그를 변경하지 않는다. 배포, 마이그레이션 적용, 스케줄 설치, 운영 그림자 기록, 소유자 전환은 각각 범위가 명시된 별도 승인이 있어야 한다.

## 전환 전에 반드시 확인할 것

- 런타임 플래그 레지스트리는 승인된 12개 행과 정확히 일치해야 한다.
- 공통·업무 어댑터 런타임 표시는 모두 버전 1이어야 한다. 등록 관련 소유자를 다룰 때는 등록 예약 알림 런타임 표시도 버전 1이어야 한다.
- 비공개 worker 중단 래치는 해제 상태여야 한다.
- worker와 독립 watchdog의 마지막 성공 heartbeat가 모두 3분 이내여야 한다.
- 그림자 모드의 실제 공급자 요청과 정규 알림함 투영은 각각 0건이어야 한다.
- 중복 외부 시도, 새 전달 결과 불명(`delivery_unknown`), 5분 초과 대기, 범위 불일치, 대상 없는 활성 규칙, 소유권 이상, 그림자 의도 불일치는 모두 0건이어야 한다.
- 비교의 `targetGeneration`은 10진 문자열인 업무 수신자 세대다. 별도 dispatch `ownerGeneration`으로 대체하면 검증 실패다.

## 고정 전환 순서

1. 할 일
2. 영어 단어 재시험
3. 전자결재
4. 전반
5. 퇴원
6. 휴보강
7. 등록 공통·예약 알림
8. 등록 전화상담 알림함
9. 등록 방문상담 즉시 알림
10. 등록 SOLAPI 고객 알림

첫 전환만 전역 그림자를 끄면서 한 소유자 플래그를 원자적으로 켠다. 그다음부터는 그림자를 계속 끈 상태에서 한 소유자씩 순서대로 전환한다. 한 단계의 관찰이 끝나기 전에는 다음 단계로 진행하지 않는다.

## 그림자 비교

아래 항목을 정규화한 뒤 비교한다.

- 업무, 이벤트, 발생 ID
- 대상 그룹, 채널, 대상 키
- 수신자 세대
- 템플릿 체크섬
- 정규화된 렌더 결과 해시

차이는 누락 이벤트, 추가 이벤트, 대상 불일치, 수신자 세대 불일치, 채널 불일치, 템플릿 불일치로 분류한다. 미리보기·fixture에서는 기존 경로와 정규 경로를 합친 실제 외부 요청이 0건이어야 한다.

## 자동 중단과 롤백

운영 전환 전 그림자 단계에서 공급자 요청이나 알림함 투영이 한 건이라도 발생하면 그림자 중단 RPC를 호출해야 한다. 모든 dispatch 플래그와 설정 UI를 끄고, 보내지 않은 작업을 취소하며, worker 중단 래치를 올린다. 자동으로 그림자를 다시 켜지 않는다.

전환 시작 뒤 worker/watchdog·스케줄·Vault·경로의 전역 장애나 업무 간 중복 시도는 전체 소유자 롤백 대상이다. 전체 소유자 롤백은 현재 켜진 모든 소유자 플래그를 한 트랜잭션에서 끄고 전역 그림자를 안전 기준인 꺼짐으로 둔다. 장애가 그림자·worker 안전을 훼손하지 않았고 모든 활성 소유자를 함께 내리는 경우에만 그림자 복구를 별도로 요청할 수 있다.

한 업무의 지연, 전달 결과 불명, 대상 없는 규칙, 범위·소유권 이상, 그림자 불일치는 부분 롤백 대상이다. 부분 롤백은 해당 소유자만 끄고 그림자를 계속 끈 상태로 유지하므로 정상인 다른 정규 소유자는 계속 동작한다.

롤백은 `pending`·`retry_wait`를 취소하고 보내기 전 `claimed`에는 취소 요청을 남긴다. `sending`, `sent`, `failed`, `canceled`, 전달 결과 불명 기록은 보존한다. 전송이 시작됐거나 공급자 참조가 있는 소유권은 기존 경로로 넘기지 않는다. 롤백 실패는 치명 경보로 기록하고 성공 heartbeat를 남기지 않으며 다음 전환을 금지한다.

## 승인 경계

- 코드/미리보기 완료는 운영 그림자나 운영 전환 승인이 아니다.
- 7일 운영 그림자는 별도 승인 후에만 시작한다.
- 실제 소유자 전환은 순서별로 다시 승인받는다.
- 운영 플래그, 비밀정보, 실제 endpoint, 공급자 본문은 코드 커밋에 넣지 않는다.
- Google Chat, Web Push, SOLAPI 실제 발송은 대상과 자격 증명이 확정된 별도 테스트 승인 없이는 통과로 기록하지 않는다.

## 보존

기존 설정·전송·공급자 상태 읽기는 정규 단독 운영을 시작한 뒤 최소 14일과 성공한 롤백 훈련 1회가 모두 끝날 때까지 유지한다. 삭제는 이 절차에 포함하지 않으며 별도의 검토 계획이 필요하다.
