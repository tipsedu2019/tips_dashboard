# 작업 3 구현 결과 보고서

상태: 수정 가능한 가짜 이력을 제거하고, 변경 불가능한 주요 사건과 정직한 이전 자료만 보여 주는 읽기 전용 등록 이력을 구현했습니다.

구현 커밋: `5667ad4`

## 구현 내용

- `RegistrationHistoryTimeline`을 등록 상세에 연결하고 과목·단계 필터와 상세 펼치기만 제공했습니다. 이력 안에는 입력·수정·삭제 동작이 없습니다.
- 변경 가능한 현재 담당자와 단계는 `현재 업무`로 분리해 사건 행위자와 섞이지 않도록 했습니다.
- 공유 예약은 참여 과목을 함께 표시하는 주요 단계 한 행으로 묶고, 예약 변경·교체·과목 제외는 해당 행의 상세 변경으로 흡수했습니다.
- 실제 SQL의 `appointment_replaced` 형태처럼 새 예약의 시간·장소가 직접 없더라도 기존 예약 앵커를 찾아 중복 행을 만들지 않습니다. 변경 전 값은 앞선 사건 사실에서만 추론합니다.
- 알림 사건, 분배, 전달, 재시도, 공급자 내부 행은 이력에서 제외했습니다.
- version-2의 사용자·시스템·마이그레이션 행위자를 구분하고, version-1의 null 행위자는 현재 담당자로 추측하지 않고 `알 수 없음`으로 표시합니다.
- 정식 사건이 없는 스냅숏은 시각을 추측하지 않고 `시간 확인 불가 · 마이그레이션`으로 표시합니다. 변경 가능한 `updatedAt`은 사건 시각으로 사용하지 않습니다.
- 실제 SQL에서 사용하는 `migration_review`, `unassigned`, `default`, `manual`, `enrollment`, `waitlisted`, `appointment_subject_deselected`를 한글 운영 용어로 표시하면서 원시 감사 값은 보존했습니다.
