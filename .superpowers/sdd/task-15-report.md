# 작업 15 구현 결과 보고서

상태: 할 일과 영어 단어 재시험의 브라우저 사후 이벤트 쓰기를 제거하고, 업무 변이와 권위 이벤트를 고정 원자 RPC로 연결했습니다.

## 구현 내용

- 할 일 생성·수정·상태 변경·댓글을 각각 고정 목적 RPC로 처리합니다.
- 댓글 원본 ID와 활동 이벤트 ID를 구분해 원본 이벤트 변환기가 정확한 원본을 찾습니다.
- 생성 실패 정리는 등록 업무 한 건, 정확한 생성 시각, 1시간 이내, 비종결·트랙 미연결 조건을 모두 만족할 때만 허용합니다.
- 영어 단어 재시험 재시도는 기존 건 종료, 이전·새 건 연결, 새 건 생성, 이벤트 기록을 한 트랜잭션에서 처리합니다.
- 인증 사용자의 `ops_tasks`·`ops_word_retests` 직접 추가·수정·삭제 권한을 회수해 고정 RPC를 우회할 수 없게 했습니다.
- 공개 활동 기록 RPC에서는 서버 변이에서만 만들 수 있는 생성·수정·상태 변경·재시험 요청 이벤트를 거절합니다.
- 모든 알림 플래그는 꺼진 상태를 유지합니다.

## 검증

- 생산자·정리·원자성 관련 집중 회귀를 통과했습니다.
- 브라우저 직접 이벤트 INSERT/DELETE 경로가 남지 않았음을 정적 검사했습니다.
- TypeScript와 대상 ESLint를 통과했습니다.
- 실제 DB pgTAP은 승인된 실행 대상이 없어 실행하지 않았고, SQL 권한·원자성 계약은 정적 회귀로 검증했습니다.
- 실제 DB 쓰기와 실제 공급자 발송은 실행하지 않았습니다.
