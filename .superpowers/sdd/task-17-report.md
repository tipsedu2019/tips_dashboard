# 작업 17 구현 결과 보고서

상태: 휴보강의 기존 설정·템플릿·전달 이력을 보존하면서 승인과 알림 소유권을 정규 경계에 연결했습니다.

## 구현 내용

- 기존 휴보강 설정과 템플릿 체크섬을 기준 변경 번호와 함께 비교합니다.
- 변경된 설정만 다시 조정하고 기존 전달·이력·발생 건 참조는 보존합니다.
- 승인·재처리·댓글 활동을 고정 RPC로 기록합니다.
- 동일 강의실·겹치는 시간의 동시 승인 요청은 강의실별 잠금을 정렬해 획득한 뒤 다시 조회하므로 둘 다 승인될 수 없습니다.
- 생성 응답이 유실된 재시도는 같은 요청 ID와 신청 내용을 재사용하고, 성공 또는 확정 실패 뒤에만 재시도 기록을 정리합니다.
- 업무 저장 성공과 기존 공급자 전달 실패를 분리해 알림 실패가 휴보강 신청을 되돌리지 않게 합니다.
- 정규 그림자와 기존 실제 경로가 같은 발생 건을 중복 발송하지 못하게 합니다.

## 검증

- 휴보강 모델·서비스·화면·어댑터 집중 회귀를 통과했습니다.
- 정규 그림자 전달은 `skipped/shadow_mode`이며 알림함에 투영하지 않음을 검사했습니다.
- 실제 DB pgTAP은 승인된 실행 대상이 없어 실행하지 않았고, 잠금·재실행·권한 계약은 정적 회귀로 검증했습니다.
- 실제 공급자 호출은 실행하지 않았습니다.
