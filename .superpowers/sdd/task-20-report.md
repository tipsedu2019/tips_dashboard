# 작업 20 구현 결과 보고서

상태: 구형 알림 계약을 안전하게 관찰·폐쇄하고 정규 작업 처리기를 운영할 수 있는 코드와 순방향 DB 계약을 구현했습니다. 원격 적용은 실행하지 않았습니다.

## 구현 내용

- Google Chat과 Web Push의 원문형 구형 진입점은 인증과 무내용 계측만 수행한 뒤 `skipped=true`로 종료하며 외부 공급자를 직접 호출하지 않습니다.
- 고정 업무 진입점은 안정된 원본 이벤트 ID만 받아 서버 변환기로 해석하고, 브라우저가 보낸 제목·본문·수신자를 신뢰하지 않습니다.
- 브리지 설치 뒤 24시간과 완결된 서울 운영일, 단일 빌드 영수증, 고정 경로 성공을 DB가 다시 확인해야만 구형 계약을 폐쇄할 수 있습니다.
- 작업 처리기와 독립 감시기의 고정 일정 계약·검증·관리 스크립트를 구현했습니다. 운영 일정은 설치하지 않았습니다.
- 그림자 외부 요청, 알림함 누수, 중복 시도, 결과 불명, 지연, 대상 없음과 소유권 이상을 운영 지표로 집계합니다.
- Google Chat과 Web Push가 5xx를 반환하면 공급자의 수락 여부를 확정할 수 없으므로 전달 결과 불명으로 보존하고 자동 재전송하지 않습니다.
- 폐쇄 뒤 그림자 증거는 활성 규칙이 있으면 자연 발생 기존·정규 비교를 요구하고, 활성 규칙이 0개인 범위만 현재 설정 요약값과 그림자 변경 번호에 묶인 별도 증거를 사용합니다.
- 구형 호환, 폐쇄, 일정, 등록 공급자 점유, 순방향 호환, 그림자 증거를 서로 다른 순방향 마이그레이션으로 분리했습니다.

## 검증

- 알림 관련 회귀 `326/326`, 업무·화면 통합 `961/961`, 전체 Node `1483/1483`을 통과했습니다.
- TypeScript와 전체 ESLint를 통과했습니다. ESLint 오류는 0건이며 이번 변경과 무관한 기존 경고 1건만 남아 있습니다.
- 운영용 Next.js 빌드와 정적 페이지 `78/78` 생성을 통과했습니다.
- 변경 공백 검사와 잠긴 기존 마이그레이션 무변경 검사를 통과했습니다.
- 독립 코드 감사에서 P0·P1이 0건임을 확인했습니다.
- 실제 PostgreSQL pgTAP은 승인된 실행 DB가 없어 실행하지 않았습니다.
- 원격 DB 쓰기, 운영 플래그 변경, 일정 설치, 실제 공급자 호출과 배포는 실행하지 않았습니다.
