# 작업 6 구현 결과 보고서

상태: 역할 기반 공통 알림 설정 API, 원자적 저장·감사·멱등성, Google Chat 연결 암호화와 안전한 호환 경계 구현 및 로컬 검증을 완료했습니다.

구현 커밋: 최종 검증 후 기록

## 구현 범위

- 관리자와 운영 담당자만 공통 알림 설정을 읽고 저장할 수 있습니다. 설정 저장은 같은 트랜잭션 안에서 설정 화면 활성 플래그를 다시 확인합니다.
- 요청 ID 잠금 뒤 업무별 공용 잠금을 획득하고, 변경 규칙을 정규 UUID 순서로 잠급니다. 모든 예상 revision을 먼저 검사하므로 하나라도 오래된 값이면 아무 규칙도 덮어쓰지 않습니다.
- 대소문자가 다른 동일 UUID 별칭도 거부해 한 요청에서 같은 규칙이 두 번 갱신되지 않게 했습니다.
- 실제 변경이 있을 때만 규칙 revision을 한 번 증가시키고, 템플릿 내용이 달라졌을 때만 새 불변 템플릿 버전을 만듭니다.
- SQL이 예약 알림의 업무·이벤트·기준 시각·서울 시간대·시간 범위를 최종 검증합니다. 허용되지 않은 템플릿 변수, 짝이 맞지 않는 중괄호, HTML·멘션·외부 URL도 브라우저 검증과 무관하게 DB에서 거부합니다.
- 꺼져 있던 Google Chat 규칙을 새로 켤 때 필요한 연결 행을 안정된 순서로 잠그고 상태를 재확인합니다. 관리팀·경영진은 해당 연결 한 개, 과목팀은 영어·수학 연결 두 개가 모두 정상이어야 합니다.
- 동일 요청 ID와 동일 내용은 커밋 결과를 재생하고, 동일 요청 ID에 다른 내용을 붙이면 거부합니다.
- 변경 전후 요약은 비밀정보 없이 감사 로그에 남기고, 변경 규칙이 있을 때만 규칙 재계산 작업을 한 건 만듭니다. 무변경 저장은 감사·재계산 작업을 만들지 않습니다.

## 런타임 플래그 안전성

- 플래그 변경은 `service_role` 전용이며 공통 런타임, 업무 어댑터 표식, 최근 3분 이내 성공 heartbeat, 등록 전용 표식을 필요한 조합대로 재확인합니다.
- 업무 플래그를 끌 때 아직 발송 전인 항목만 취소하거나 취소 요청으로 전환하고, 발송 중·발송 완료·결과 불명·실패 기록은 보존합니다.
- 일반 등록, 전화상담, 방문상담, SOLAPI를 서로 다른 소유 범위로 분리했습니다. 한 전용 플래그를 꺼도 다른 전용 경로나 일반 등록 발송은 취소되지 않습니다.

## Google Chat 연결 안전성

- 연결 URL은 32바이트 Base64 키를 사용하는 AES-256-GCM `v1` 봉투로 암호화합니다.
- 브라우저 응답에는 연결 상태, 문자열 revision, 고정 형식 마스크, 검증 시각과 닫힌 오류 코드만 포함합니다.
- 연결 교체·해제·검증 예약 RPC는 브라우저 세션에서 직접 호출할 수 없습니다. 서버가 인증된 관리자 ID를 명시해 `service_role`로 호출하고 DB가 관리자 권한을 다시 확인합니다. 검증 결과 기록은 예약 원장을 capability로 사용해, 예약 뒤 역할이 바뀌어도 이미 발생한 provider 결과를 안전하게 종결합니다. 예약이 없거나 다른 actor가 사용하면 현재 관리자 여부를 다시 확인해 거부합니다.
- 기존 마스크가 비어 있는 `legacy_active` 행은 서버가 저장된 URL을 엄격히 검증한 뒤 즉석에서 마스킹합니다. 잘못된 기존 값은 평문을 노출하지 않고 `configuration_error`로 표시합니다.
- 연결 해제 상태는 기존 Google Chat 발송 경로와 등록 상담 발송 경로 모두에서 환경변수 대체값보다 우선합니다. 예전 환경변수가 남아 있어도 해제된 연결이 되살아나지 않습니다.
- 기존 `/api/google-chat` 웹훅 수정 화면도 암호화 CAS 경로를 사용하며, 평문과 암호문을 한 트랜잭션에서 함께 갱신합니다.
- 백필은 기본 dry-run이고 `--apply`에서만 씁니다. 실제 쓰기는 채널·상태·revision·암호문 부재·평문 SHA-256 지문을 한 트랜잭션에서 비교한 뒤 반영합니다.
- 연결 검증 예약에는 2분 만료 시각을 저장합니다. 프로세스 중단은 만료로, 검증 중 연결 교체·해제는 superseded로 원장을 종료하며 현재 연결을 덮어쓰지 않습니다.
- 만료·superseded 결과는 성공 응답으로 가장하지 않고 API에서 409로 반환합니다. 예약 뒤 관리자 역할이 바뀌어도 이미 발생한 provider 결과는 서버가 안전하게 종결할 수 있습니다.
- 실제 Google Chat 검증은 관리자가 확인한 별도 동작에서만 고정 테스트 문구로 가능하며, 자동 테스트에서는 외부 네트워크를 전부 차단했습니다.

## 외부 상태

- Supabase 원격 마이그레이션 적용, 원격 데이터 변경, 런타임 플래그 변경, 실제 provider 호출, 배포는 수행하지 않았습니다.
- 실제 pgTAP DB 실행은 승인된 local/preview DB에 작업 5·6 마이그레이션을 적용하는 배포 전 검증 단계에 남겨 두었습니다. SQL source 계약과 pgTAP 패킷은 완료했지만 DB에서 실행한 것으로 보고하지 않습니다.
- 작업 트리 전용 개발 서버는 `http://localhost:3001`에서 계속 실행 중입니다.
