# Repository Instructions

## Design System Principles

1. 디자인은 어떻게 보이는가가 아니라, 어떻게 작동하는가다. 모든 UI 판단은 장식보다 사용자의 다음 행동을 더 빠르고 명확하게 만드는지를 기준으로 한다.

2. 단순함은 궁극의 정교함이다. 단순화는 기능을 덜어내는 일이 아니라, 사용자가 본질에 집중하도록 복잡함을 제거하는 일이다.

3. 최고의 UX/UI에는 설명이 필요 없다. 요약, 설명, 보조카드 없이도 직관적으로 바로 알 수 있고, 가장 최적의 동선으로 편리하게 사용할 수 있게 설계한다.

4. 미니멀리즘은 불필요한 장식과 버튼을 제거하고, 해당 화면의 핵심 기능만 남기는 운영 원칙이다. 남아 있는 요소는 모두 업무 흐름에 직접 기여해야 한다.

5. 직관성은 사용자가 설명서 없이 바로 사용할 수 있는 상태를 뜻한다. 이름, 위치, 상태, 다음 행동이 화면 안에서 자연스럽게 드러나야 한다.

6. 기술의 발전은 생각보다 빠르다. 바퀴를 새로 다시 만들지 말라. 업그레이드된 기술을 깊이 탐색하고, 선택하고, 응용해서 우리에게 적용해라.
